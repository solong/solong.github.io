Case Study: Schema.TF

Summary: SchemaTF provides an interesting use case for change monitoring, real-time notifications, and data analysis of long term schema history for digital asset definitions. Deliverables included a reproducible cloud-first implementation in AWS using serverless computing to minimize running costs.

Background

According to Juniper Research the current Video Game loot box market is estimated to be worth $30 billion worldwide.

Digital inventory assets in popular video games such as Dota 2 and Team Fortress 2 derived from these boxes carry significant individual market value. If the item is tradeable the secondary market value can be immense in certain circumstances. If the item is non-tradable it can make the resale value of the account holding the asset increase.

The release of a new digital asset typically leads to common price patterns. A brief price surge as traders attempt to assess the market value, followed by a dive in price for those assets that fail to gain interest. Key to avoiding losses in this market is being able to make an assessment on the nature of the item in a timely manner.

As a guide, some of the most high demand, short supply assets, can fetch prices of $30,000 each.

Schema.TF is designed to monitor for schema changes to provide timely information to traders/subscribers about key changes to the market inventory schemas. A schema is a catalog of digital asset definitions, describing the item and its expected attributes. SchemaTF catalogs the changes over time, providing analysis on the differences between the schemas, as well as providing enhanced metadata for each digital asset.

Technical Brief

Summary To ensure when a change occurs to Valve's 'Team Fortress' schema, the change is recorded for later analysis, while users are notified immediately.

  • Provide a continuous monitoring mechanism for schema changes
  • Strictly adhere to API limits provided in the Terms and Conditions of Steam's service
  • Capture changes and protect against future data loss
  • Notify users through publicly available mediums (Twitter etc)
  • Allow analysis of schema deltas from historical dataset
  • Provide authenticated intermediary data API with appropriately rate limits
  • Web Interface for management of backend services and display of key information
  • Integration with Steam Authentication
  • Item desirability score based on key attributes to alert to potential high value assets

Technologies

Java, AWS Lambda, AWS Step Function, Twitter API, Spring, Angular, AWS S3
